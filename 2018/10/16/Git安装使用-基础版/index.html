连接远程仓库

一般在建立完本地仓库后，会将本地仓库连接到远程仓库，如Github、GitLab、Coding等。这样加可以将仓库同步到云端，也方便与其他人共享。

此时需要一个Github的账号，注册账号可以去Github的官网进行注册。注册时使用的用户名和邮箱和上面登录Git的一致。

这时，本地的Git仓库与Github还没法传输，因为使用的是SSH加密，需进行一下设置。

  • 创建SSH Key，在用户主目录下，看看有没有.ssh目录，如果有，再看看该目录下有没有id_rsa和id_rsa.pub这两个文件，如果已有，可直接跳到下一步。如果没有，则打开终端或者Git Bash，创建SSH Key
  • 登录自己的Github，点击右上角的Setting选项。
  • 点击SSH and GPG keys
  • 点击 New SSH key后，输入Title和Key.Title随便取，便于记忆就好，Key是你电脑中刚才生成的id_rsa.pub里的内容，直接全选复制到这里的Key中。
  • 点击Add SSH Key,就完成了SSH的设置。这样设置是为了线上线下的验证。Github的公钥可以添加多个。

本地上传到Github

首先在Github上新建一个空白仓库，取名和刚才本地的仓库一致。点击自己Github主页右上角的+ 然后点击New respository，Repository name和本地仓库名一致，不要勾选“Initialize this repository with a README”，直接点击 Create repository，如果是在云端创建仓库，然后同步到本地，则可以勾选。
